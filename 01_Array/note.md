# Array（陣列）

## 1. Definition（定義）

Array（陣列）是一種線性資料結構，用於儲存**相同資料型態**的元素，
並將所有元素存放於**連續的記憶體位置**中。
由於記憶體連續，Array 可以透過索引（index）在 **O(1)** 時間內存取資料。

---

## 2. Characteristics（特性）

| 特性 | 說明 |
|----|----|
| 固定大小 | 宣告後大小不可變 |
| 連續記憶體 | 提升快取命中率 |
| 索引存取 | 直接存取元素 |
| 高效率 | 存取時間 O(1) |

---

## 3. Time & Space Complexity（時間與空間複雜度）

### 時間複雜度（Time Complexity）

| 操作 | Big-O | 說明 |
|----|----|----|
| 存取（Access） | O(1) | 透過 index 直接存取 |
| 搜尋（Search） | O(n) | 線性搜尋 |
| 插入（Insert） | O(n) | 需搬移元素 |
| 刪除（Delete） | O(n) | 需補齊空缺 |

---

### 空間複雜度（Space Complexity）

| 情況 | 空間複雜度 |
|----|----|
| Array 本身 | O(n) |
| 額外空間 | O(1) |

Array 不需要額外指標，因此空間使用效率高。

---

## 4. Advantages & Disadvantages（優缺點）

### 優點
- 存取速度快
- 結構簡單
- 記憶體連續、效率高

### 缺點
- 大小固定，不具彈性
- 插入與刪除成本高

---

## 5. Use Cases（使用情境）

- 成績表
- 靜態資料查詢
- 排序後資料（Binary Search）
- 圖像像素資料

---

## 6. Common Problems for Beginners（初次學習可能遇到的問題）

在初次學習 Array（陣列）時，常會遇到以下幾個問題：

### 1. 陣列大小固定，無法動態調整
剛開始十，可能會誤以為陣列可以像變數一樣隨時新增元素，  
實際上陣列在宣告時就必須指定大小，之後無法再改變，  
若資料數量預估錯誤，可能導致陣列空間不足或浪費記憶體。

---

### 2. 索引越界（Index Out of Bounds）
由於陣列索引是從 `0` 開始，  
初學者容易在迴圈或存取時發生「超出範圍」的錯誤，例如：

- 存取 `arr[size]`
- 迴圈條件寫成 `i <= size`

這類錯誤可能造成程式當掉的結果。

---

### 3. 插入與刪除操作的效能問題
在陣列中間插入或刪除元素時，  
需要將後面的資料全部往前或往後搬移，  
初學者往往忽略這個成本，導致效能變差。

---

### 4. 混淆「存取快」與「搜尋快」
Array 的「存取（Access）」是 O(1)，  
但「搜尋（Search）」仍然是 O(n)。  

---

### 5. 不清楚什麼時候該用 Array
剛學資料結構時，容易「所有資料都用 Array 解決」，  
但在需要頻繁插入、刪除或資料量不固定時，  
Array 並不是最好的選擇。
